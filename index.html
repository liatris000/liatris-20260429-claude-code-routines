⚡Claude Code Routinesとは

2026年4月14日、Anthropicがリサーチプレビューとして公開した新機能です。 「何をしてほしいか」を自然言語で書くだけで、Claudeがスケジュールやイベントに応じてAnthropicのクラウド上で自律実行します。 ローカルマシンを起動していなくても動き続けるのがポイントです。

🕐

Schedule

cronライクに毎時・毎日・毎週など定時実行。夜間バッチや朝のデイリーレポートに最適。

🐙

GitHub Event

PR作成・push・リリースなどWebhookイベントで即時起動。PRレビューの自動化に使う。

🔌

API

専用エンドポイントへのPOSTで任意タイミングに起動。外部サービスとの連携に。

🛠セットアップ手順

1. Routineの設定ファイルを作成

リポジトリルートに .claude/routines/ ディレクトリを作り、YAMLで定義します。

.claude/routines/pr-review.yml YAML 
name: PR自動レビュー
description: PRが開かれたとき、コードの品質・セキュリティ・パフォーマンスを自動レビュー

triggers:
  github:
    events: [pull_request.opened, pull_request.synchronize]
    filters:
      draft: false               # ドラフトPRはスキップ
      base_branch: main       # main向けのPRのみ

prompt: |
  あなたは経験豊富なコードレビュアーです。
  以下の観点でPRをレビューし、PRコメントとして投稿してください。

  ## レビュー観点
  1. **バグ・ロジックエラー**: 明らかな誤り・エッジケースの漏れ
  2. **セキュリティ**: SQLインジェクション・XSS・認証・認可の問題
  3. **パフォーマンス**: N+1クエリ・不要な計算・メモリリーク
  4. **可読性**: 命名・コメント・複雑すぎる実装
  5. **テスト**: カバレッジが不十分な部分の指摘

  問題がない場合は「LGTM ✅」と投稿。
  重要な問題が見つかった場合は「Changes requested」をリクエスト。

connectors:
  github:
    repo: {{ github.repository }}
    permissions: [pull_requests:write, contents:read]

2. Claude Code CLIで登録

Terminal bash 
# Routineを登録
claude routines add .claude/routines/pr-review.yml

# 登録確認
claude routines list

# 手動テスト実行
claude routines run pr-review --dry-run
💡 dry-run フラグをつけると、実際にコメントは投稿されずに実行内容だけ確認できます。本番適用前に必ず試しましょう。

🔍自動レビューの実際の出力例

以下は実際にRoutinesがPRに投稿したコメントのサンプルです。

🔀 feat: ユーザー認証にJWTを導入 Open #47 by @dev-tanaka
C claude-routines bot 2分前

PRのレビューを完了しました。以下の点をご確認ください。

⚠️ [セキュリティ] JWTシークレットのハードコード (auth/jwt.ts:L14)

- const SECRET = "my-super-secret-key"; + const SECRET = process.env.JWT_SECRET; + if (!SECRET) throw new Error("JWT_SECRET is not set");

シークレットキーが直書きされています。process.env.JWT_SECRET を使用してください。

⚠️ [セキュリティ] トークン有効期限が長すぎる (auth/jwt.ts:L28)

- expiresIn: "365d" // 1年 + expiresIn: "7d" // 推奨: 7日以内

✅ [良い点] リフレッシュトークンの実装が適切です。アクセストークンとリフレッシュトークンを分離しているのは良い設計です。

💡 [提案] テストカバレッジ
トークン期限切れ・無効トークンのテストケースが未実装です。セキュリティに関わる部分のため追加を推奨します。

.claude/routines/ci-autofix.yml YAML 
name: CI失敗自動修正

triggers:
  github:
    events: [check_run.completed]
    filters:
      check_run.conclusion: failure

prompt: |
  CIが失敗しました。失敗ログを確認し、
  Lintエラーや型エラーなど自動修正可能なものを修正して
  PRにコミットしてください。
  手動対応が必要な場合はPRにコメントで内容を説明してください。

connectors:
  github:
    repo: {{ github.repository }}
    permissions: [contents:write, pull_requests:write]
.claude/routines/on-demand.yml YAML 
name: オンデマンド分析

triggers:
  api:
    auth: bearer

prompt: |
  受け取ったパラメータに基づいて分析を実行してください:
  - target_branch: {{ inputs.target_branch }}
  - analysis_type: {{ inputs.analysis_type }}

connectors:
  github:
    repo: {{ github.repository }}
APIから呼び出す例 bash 
curl -X POST https://api.anthropic.com/v1/routines/on-demand/trigger \
  -H "Authorization: Bearer $ANTHROPIC_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "inputs": {
      "target_branch": "feature/new-auth",
      "analysis_type": "security"
    }
  }'

🔄PR自動レビューのフロー

👨‍💻 開発者がPRを作成
→
🐙 GitHub Webhook発火
→
⚡ Routine起動
→
🤖 Claudeがdiffを解析
→
💬 PRにコメント投稿

GitHubのWebhookがRoutineをリアルタイムで起動するため、PRを作った数分以内にレビューコメントが届きます。 ローカルマシンが起動していなくても動作し、深夜・休日のPRにも即座に対応します。
